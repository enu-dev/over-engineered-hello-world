## 開発の目的・解決した課題

> 広報チームが「なぜ作ったか」を投稿に反映するための素材

「Hello World」はプログラミングの最初の一歩でありながら、最もシンプルな例として扱われる。これを意図的に「過剰に凝った」実装にすることで、技術力のデモとギャップコメディを同時に成立させた。

**課題→解決:**
- 「技術力があることを見せたいが、大げさに語りたくない」→ 一番シンプルなお題（Hello World）を一番複雑に実装することで、技術は行動で見せる
- 「AIに作らせたと思われる」→ 作るプロセス・こだわりのディテール（ローダーのテキスト、コナミコード等）が「人が作った感」を演出

## 技術的ハイライト（バズに使える要素）

優先度が高い順に：

1. **ファイル構成：index.html + css/style.css + js/script.js（合計 542 行）**
2. **ライブラリ一切なし・バニラ JS のみ**（Canvas 2D、requestAnimationFrame、CSS animations）
3. **ダブルクリックで動く・環境構築ゼロ**（GitHub Pages でも file:// でも動作）
4. **80 個のパーティクルが Canvas で動く**（コンステレーション線・マウス反発・速度減衰）
5. **偽ブートシーケンス**（「React: 未インストール（意図的）」がウケる）
6. **スマホでも動作**（clamp() フォント、touch イベント対応）
7. **コナミコード搭載**（↑↑↓↓←→←→BA）

## 詰まったこと・失敗したこと（コンテンツに使える素材）

> 「正直に話してる人」感を演出するための素材

1. **CSS グリッチに `clip-path` を使ったら Safari で動かなかった** → `text-shadow` の色ずれ + `skewX` に全面切り替え。Safariの互換性問題はやっぱりある。
2. **CSS アニメーションの再発火問題** → `.glitching` クラスを外して再付与するだけではアニメーションが再起動しない。`void el.offsetWidth` でリフローを強制する必要があった（これ知らなかった）
3. **`cursor: none` を `body` に指定してもSafariで効かない** → `*, *::before, *::after { cursor: none; }` と全要素に適用して解決
4. **グリッチストーム中に `isGlitching` フラグが邪魔をした** → 連続グリッチを意図的に起こすためにフラグを外からリセットする処理を追加（設計上の妥協点）

## えぬ向けのGitHub Pages公開手順書

1. GitHub で新しいリポジトリを作成する
   - リポジトリ名：`over-engineered-hello-world`
   - 公開設定：**Public**
   - README の初期化：**しない**（後でアップロードする）

2. 成果物ファイルをすべてアップロードする（フォルダ構成を維持すること）
   ```
   index.html
   css/style.css
   js/script.js
   README.md
   LICENSE
   ```
   ※ `css/` フォルダと `js/` フォルダごとアップロードすること（フォルダ構造が壊れると CSS/JS が読み込まれない）

3. GitHub Pages を有効化する
   - Settings → Pages → Source: "Deploy from a branch"
   - Branch: `main` / `(root)`
   - Save をクリック

4. 数分後、以下のURLで公開される
   - `https://[えぬのGitHubユーザー名].github.io/over-engineered-hello-world/`

5. README.md の「デモURL」欄に上記URLを記入してコミット

6. リポジトリの説明文（About）に以下を設定：
   - Description：`「Hello World」を表示するだけなのに全力を注いだ結果`
   - Website：GitHub Pages URL

7. Topics タグを設定する：
   - `javascript`
   - `html-css-javascript`
   - `vanilla-js`
   - `canvas`
   - `claude-code`
   - `hello-world`

8. 公開 URL とリポジトリ URL を CEO（高橋）に共有する
